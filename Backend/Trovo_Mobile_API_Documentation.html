Trovo Mobile App - API Documentation

This document contains all the necessary API endpoints for the Trovo Mobile Application (Students Side). It is intended for the mobile app development team.

Base URL: https://your-domain.com/api
Authentication: All endpoints (except Register and Login) require a Bearer Token in the headers:
Authorization: Bearer {access_token}
Accept: application/json

1. Authentication

POST /register

Register a new student account from the mobile app.

Request Body

ParameterTypeDescription
nameStringStudent's full name (Required)
student_codeStringUnique university student code (Required)
phone_numberStringContact number (Optional)
passwordStringAccount password (min: 6 chars) (Required)

Success Response (201 Created)

{
  "status": "success",
  "message": "تم إنشاء حسابك بنجاح! يمكنك استخدام التطبيق الآن.",
  "access_token": "1|xyz...",
  "token_type": "Bearer",
  "user": {
    "id": 1,
    "name": "John Doe",
    "student_code": "20231001",
    "phone_number": "01000000000"
  }
}
POST /login

Login to the app using the student code and password.

Request Body

ParameterTypeDescription
usernameStringStudent code (Required)
passwordStringAccount password (Required)

Success Response (200 OK)

{
  "status": "success",
  "access_token": "2|xyz...",
  "token_type": "Bearer",
  "user": {
    "id": 1,
    "name": "John Doe",
    "student_code": "20231001",
    "phone_number": "01000000000"
  }
}
GET /user Auth Required

Get the currently authenticated student's profile data.

Success Response (200 OK)

{
  "status": "success",
  "data": {
    "id": 1,
    "name": "John Doe",
    "student_code": "20231001",
    "is_approved": 0,
    ...
  }
}
POST /logout Auth Required

Logout and invalidate the current access token.

Success Response (200 OK)

{
  "status": "success",
  "message": "تم تسجيل الخروج بنجاح"
}

2. Phone Usage Data

POST /phone-usage Auth Required

Submit daily phone usage statistics from the student.

Request Body

ParameterTypeDescription
daily_usage_hoursNumericHours per day (0-24) (Required)
screen_time_before_bedNumericHours before bed (0-24) (Required)
phone_checks_per_dayIntegerNumber of times phone is checked (Required)
apps_used_dailyIntegerNumber of apps used (Required)
time_on_social_mediaNumericHours on social media (0-24) (Required)
time_on_gamingNumericHours on gaming (0-24) (Required)
phone_usage_purposeString'Social Media', 'Gaming', 'Education', 'Other' (Required)
weekend_usage_hoursNumericWeekend usage hours (0-24) (Required)

Success Response (201 Created)

{
  "status": "success",
  "message": "تم تسجيل بيانات الاستخدام بنجاح للطالب: John Doe",
  "data": { ... }
}
GET /phone-usage Auth Required

Get a paginated history of the student's phone usage submissions.

Success Response (200 OK)

{
  "status": "success",
  "user": "John Doe",
  "data": {
    "current_page": 1,
    "data": [ { ... }, { ... } ],
    "total": 5
    ...
  }
}

3. Questionnaire

POST /questionnaire Auth Required

Submit student's psychological and lifestyle questionnaire answers.

Request Body

ParameterTypeDescription
genderString'male', 'female', 'other' (Required)
sleep_hoursNumeric(0-24) (Required)
academic_performanceNumeric(0-100) (Required)
social_interactionsNumeric(0-10) (Required)
exercise_hoursNumeric(0-24) (Required)
anxiety_levelNumericMinimum 0 (Required)
depression_levelNumericMinimum 0 (Required)
self_esteemNumeric(0-100) (Required)
time_on_educationNumeric(0-24) (Required)

Success Response (201 Created)

{
  "status": "success",
  "message": "Questionnaire recorded successfully for John Doe",
  "data": { ... }
}
GET /questionnaire Auth Required

Get a paginated history of the student's questionnaire submissions.

4. Diagnosis (AI Powered)

POST /diagnosis/generate Auth Required

Triggers the AI to generate a diagnosis based on the latest phone usage and questionnaire data. Automatically tracks progress upon success.

Request Body

No body required. Uses existing database records.

Success Response (201 Created)

{
  "status": "success",
  "message": "Diagnosis generated successfully.",
  "data": {
    "addiction_level": "High",
    "brainrot_stage": "Stage 2",
    "analysis_intro": "...",
    "top_factors": [...],
    "recommendations": [...]
  }
}
GET /diagnosis-history Auth Required

Get a paginated list of all past diagnoses for the student.

5. Progress Tracker

GET /progress Auth Required

Get a summary of the student's progress (improvement or decline).

Success Response (200 OK)

{
  "status": "success",
  "user": "John Doe",
  "data": {
    "status": "improving",
    "details": "..."
  }
}
GET /progress/history Auth Required

Get a detailed paginated history of the student's progress over time.

6. Mini Games (Brain Training)

POST /mini-game Auth Required

Submit the results of a mini-game played by the student.

Request Body

ParameterTypeDescription
game_typeString'memory', 'reaction', 'focus', 'pattern' (Required)
scoreIntegerPoints scored (Required)
reaction_time_msIntegerIn milliseconds (Optional)
accuracy_percentageNumeric(0-100) (Optional)
difficulty_levelString'easy', 'medium', 'hard' (Required)
duration_secondsIntegerTime spent playing (1-3600) (Required)

Success Response (201 Created)

{
  "status": "success",
  "message": "تم تسجيل نتيجة اللعبة بنجاح.",
  "data": { ... }
}
GET /mini-game/history Auth Required

Get a paginated history of all mini-games played.

GET /mini-game/stats Auth Required

Get aggregated statistics (average score, best score, etc.) grouped by game type.
